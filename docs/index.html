ctrlPNL

Floating debug panels for the browser. Drop in one script tag, write a config object — get a fully interactive GUI with sliders, color pickers, toggles, folders, tabs, and more. No build step. No npm. No framework.

What is cntrlPNL?

cntrlPNL is a zero-dependency, drop-in control panel library for browser environments. It renders a floating panel UI that you configure declaratively with a plain JavaScript object. Every control binds to a reactive state proxy — read from it and write to it anywhere in your code.

It was built for creative coding, WebGL projects, and anything where you want to tweak parameters in real time without reaching for a full GUI framework.

⚡
Zero setup

One script tag. No npm install, no bundler, no config files. Works in a plain HTML file.

🔁
Reactive state

State is a Proxy object. Write to it in code and the panel updates. Move a slider and your code sees it instantly.

🎛️
13 control types

Sliders, toggles, color pickers, selects, steppers, text inputs, XY pads, folders, tabs, and more.

🎨
Themeable

One accent color per panel drives everything. Swap it out and the whole panel recolors.

When to use it

cntrlPNL is a great fit when you need to:

  • Tweak shader uniforms or WebGL parameters without hardcoding values
  • Build a live preview tool for a creative project
  • Add debug controls to a canvas animation
  • Prototype UI interactions with adjustable parameters
  • Replace a pile of <input type="range"> elements with a proper panel

It's not a replacement for a full UI framework — it's a utility panel you wire up alongside your main code.

getting started

Installation

cntrlPNL has no dependencies and no build step. Getting it running takes about 30 seconds.

Script tag

The simplest way. Download cntrlpnl.js and include it in your HTML:

<script src="cntrlpnl.js"></script>

Place it anywhere in <head> or before your closing <body> tag. The library wraps itself in an IIFE so it won't pollute your global scope beyond window.pnlStates and reading window.pnlConfigs.

Order matters. Set window.pnlConfigs before or after the script — cntrlPNL waits for DOMContentLoaded before initialising, so either order works.

From a CDN

If your project is already hosted, you can link straight to the raw file on GitHub:

<script src="https://raw.githubusercontent.com/zcy-codes/cntrlpnl/main/cntrlPNL.js"></script>

Note: Raw GitHub URLs are not a CDN — they're rate limited and may be slow. For production use, serve the file yourself or use a proper CDN like jsDelivr.

Verify it's working

After including the script, paste this in your browser console to confirm the library loaded:

window.pnlConfigs = [{ ui: { title: 'test' }, controls: [
  { type: 'slider', key: 'x', default: 50 }
]}];

You should see a small panel appear in the top-right corner of the page. If nothing appears, check that cntrlpnl.js is loading correctly in the Network tab of your browser devtools.

getting started

Quickstart

Get a panel running in under a minute, then wire it into your code.

Your first panel

Set window.pnlConfigs to an array of panel config objects. Each config produces one panel.

<script src="cntrlpnl.js"></script>
<script>
  window.pnlConfigs = [
    {
      ui: {
        title:  'scene',
        accent: '#00aaff',
      },
      controls: [
        { type: 'slider', key: 'speed',   label: 'speed',   default: 50,    min: 0, max: 100 },
        { type: 'toggle', key: 'visible', label: 'visible', default: true },
        { type: 'color',  key: 'tint',    label: 'tint',    default: '#ff4400' },
      ]
    }
  ];
</script>

This gives you a panel named "scene" with a slider, a toggle, and a color picker — all bound to a reactive state object.

Reading state

After the panel initialises, each panel's state is available at window.pnlStates[index]. Reading a property gives you the current value.

const s = window.pnlStates[0];

s.speed;    // → 50 (slider current value)
s.visible;  // → true (toggle state)
s.tint;     // → '#ff4400' (hex string)

Writing state

Writing to a state property updates the panel control in real time — the slider moves, the toggle flips, the color swatch changes.

const s = window.pnlStates[0];

s.speed = 75;           // slider jumps to 75
s.visible = false;      // toggle turns off
s.tint = '#00ff88';     // swatch updates

This is a Proxy. The state object is wrapped in a JavaScript Proxy, which intercepts all writes and triggers the panel to update. You can pass it around, store references to it, and write to it from anywhere — it behaves like a plain object.

Using state in a render loop

Reading from pnlStates inside a loop is the typical pattern for creative coding. Use the optional chaining guard (?.) to handle the moment before the panel has initialised:

function render() {
  const s = window.pnlStates?.[0];
  if (s) {
    mesh.rotation.y  += s.speed * 0.01;
    mesh.visible      = s.visible;
    scene.background  = new THREE.Color(s.tint);
  }
  requestAnimationFrame(render);
}

render();

Since pnlStates is read every frame, any change to the panel is reflected immediately with no extra event wiring.

Controls

Every control takes at minimum a type, a key, and a default. Most accept a label and an optional info tooltip string.

slider

Full-width track with a label and number readout. The number is scrubable — drag it up or down to change the value, or click it to type an exact number. Decimal step values automatically format the readout to the right number of decimal places.

live demo
{
  type:    'slider',
  key:     'exposure',
  label:   'exposure',
  default: 1.0,
  min:     0,
  max:     2,
  step:    0.01,
  info:    'scene exposure multiplier'
}
proptypedefaultdescription
keystring—Property name on the state proxy
labelstringkeyText shown above the slider
defaultnumber—Initial value
minnumber0Minimum bound
maxnumber100Maximum bound
stepnumber1Increment — decimal steps auto-format the readout
infostring—Tooltip shown on label hover

Scrub interaction

The number readout supports two interactions: drag (mousedown + move up/down) to scrub the value, and click (no movement) to focus the field and type an exact value. Pressing Enter or clicking away commits the typed value and clamps it to the min/max range.

toggle

An on/off switch that binds to a boolean state key. Clicking flips the value instantly. You can also write to the state proxy to flip it from code.

live demo
{ type: 'toggle', key: 'wireframe', label: 'wireframe', default: false }
proptypedefaultdescription
keystring—Property name on the state proxy
labelstringkeyDisplay label
defaultbooleanfalseInitial value
infostring—Tooltip on label hover

Programmatic control

// flip from code
window.pnlStates[0].wireframe = true;

// toggle
const s = window.pnlStates[0];
s.wireframe = !s.wireframe;

color

Compact hex input by default. Add a modes array to expand it into a full picker with a hue bar, saturation/brightness canvas, RGB channel inputs, and an opacity slider.

live demo
{
  type:      'color',
  key:       'bg',
  label:     'background',
  default:   '#1a1a2e',
  modes:     ['slider', 'box', 'rgb', 'opacity'],
  onOpacity: val => { myAlpha = val; }
}

onOpacity callback

Opacity is separate from the hex value because hex doesn't encode alpha. The onOpacity callback fires whenever the alpha slider moves, receiving a number between 0 and 1. Store it yourself and use it however your rendering code needs it.

let alpha = 1.0;

{
  type: 'color', key: 'fill', default: '#ff0000',
  modes: ['slider', 'opacity'],
  onOpacity: v => { alpha = v; }
}

// use in canvas rendering:
ctx.globalAlpha = alpha;
ctx.fillStyle   = s.fill;

Increment and decrement buttons with a value display. Good for integer values or anything where you want discrete steps rather than a continuous range — layer counts, quality presets, iteration depths.

point2d

An XY pad with a draggable crosshair handle. The X and Y coordinate inputs above the pad are also scrubable — drag up/down to adjust, click to type. State value is always an { x, y } object.

live demo

  yRange:  [-1, 1],

Reading and writing

const s = window.pnlStates[0];

// read individual axes
s.origin.x;  // → 0
s.origin.y;  // → 0

// write — handle moves to new position
s.origin = { x: 0.5, y: -0.3 };

Write the full object. Assigning to s.origin.x alone won't trigger a panel update — the proxy only intercepts top-level property writes. Always assign a new { x, y } object.

button / folder / tabs

Structural controls that don't bind to state keys. They organize the panel, trigger actions, or add navigation.

live demo

button

Full-width action button. onClick receives the reactive state proxy as its first argument, so you can read and write state directly.

{
  type:    'button',
  label:   'reset',
  onClick: s => {
    s.speed   = 50;
    s.visible = true;
    s.tint    = '#ff4400';
  }
}

text

A section label and visual separator. No state binding — purely for organizing the panel into named groups.

{ type: 'text', label: 'rendering' }

folder

A collapsible group. Click the folder header to expand or collapse. The panel reflows its height smoothly. Folders can be nested.

{
  type:     'folder',
  label:    'post fx',
  children: [
    { type: 'slider', key: 'bloom',   default: 0,   min: 0, max: 1, step: 0.01 },
    { type: 'slider', key: 'sharpen', default: 0.2, min: 0, max: 1, step: 0.01 },
  ]
}

tabs

Tabbed sections. Each tab has its own children array. Switching tabs animates with a slide direction based on which tab you came from.

{
  type: 'tabs',
  tabs: [
    { label: 'light',  children: [/* controls */] },
    { label: 'shadow', children: [/* controls */] },
    { label: 'camera', children: [/* controls */] },
  ]
}

link & popup

// opens a URL in a new tab
{ type: 'link', label: 'docs', url: 'https://example.com' }

    title:   'cntrlPNL',

guide

State & reactivity

How the reactive state proxy works, how to use multiple panels, and patterns for integrating state into your codebase.

The state proxy

Each panel's state is wrapped in a JavaScript Proxy. This lets cntrlPNL intercept writes and update the UI without any event system or subscription API on your end.

const s = window.pnlStates[0];

s.speed = 75;  // proxy intercepts this → slider updates
s.speed;        // → 75 (plain property read)

The proxy is transparent — you don't need to call any methods or use any special syntax. Treat it like a plain object.

Top-level writes only. The proxy intercepts writes to direct properties of the state object. Writing to nested object properties (like s.origin.x = 0.5) bypasses the proxy and won't update the panel. For point2d, always assign a new { x, y } object.

Multiple panels

Pass multiple configs in the pnlConfigs array. Each produces its own panel, stacked vertically on the right side of the screen, with its own state index.

window.pnlConfigs = [
  {
    ui: { title: 'scene', accent: '#00ff88' },
    controls: [/* ... */]
  },
  {
    ui: { title: 'post fx', accent: '#ff6600', collapsed: true },
    controls: [/* ... */]
  },
];

// each panel has its own state index
window.pnlStates[0].speed = 10;
window.pnlStates[1].bloom = 0.4;

Render loop pattern

The most common pattern for creative coding — read state on every frame. Because the proxy reads are just property accesses, this has effectively zero overhead.

function tick(dt) {
  const s = window.pnlStates?.[0];
  if (!s) return;

  mesh.rotation.y  += s.rotSpeed * dt;
  mesh.scale.setScalar(s.scale);
  material.color.set(s.color);
  material.wireframe = s.wireframe;
}

const loop = ts => {
  tick((ts - last) / 1000);
  last = ts;
  requestAnimationFrame(loop);
};
let last = 0;
requestAnimationFrame(loop);

Initialisation timing

cntrlPNL initialises on DOMContentLoaded. If your script runs before that event fires, set window.pnlConfigs first and cntrlPNL will pick it up when it initialises. If your script runs after, you can also call the internal builder directly.

// safe in any order — cntrlPNL checks pnlConfigs on DOMContentLoaded
window.pnlConfigs = [{ /* ... */ }];

// alternatively, wait for the DOM yourself:
document.addEventListener('DOMContentLoaded', () => {
  window.pnlConfigs = [{ /* ... */ }];
});
guide

Theming

cntrlPNL panels have a fixed dark base and one accent color per panel. The accent drives every highlighted element.

Accent color

Set ui.accent (new format) or theme.accent (legacy format) to any hex color. It's applied as a CSS custom property on the panel element, so it cascades to all controls inside.

ui: { accent: '#ff6600' }  // orange
ui: { accent: '#a855f7' }  // purple
ui: { accent: '#00d4ff' }  // cyan
ui: { accent: '#00ff88' }  // green (default)
ui: { accent: '#f43f5e' }  // red

The accent color affects: slider thumbs and track fill, toggle background when on, tab underlines, folder border, input focus border, button hover state, and the panel title text.

Keyboard toggle

Assign any key to collapse and expand the panel. The key is matched against event.key, so use lowercase single characters or key names like 'Escape'.

ui: { closeKey: 'h' }  // press h anywhere on page to toggle
ui: { closeKey: '`' } // backtick (common in game UIs)

Start collapsed

Useful when you want the panel available but out of the way by default.

ui: { collapsed: true, closeKey: 'h' }

Per-panel accent colors

Each panel in a multi-panel setup gets its own accent color. This makes it easy to visually distinguish them at a glance.

window.pnlConfigs = [
  { ui: { title: 'scene',   accent: '#00ff88' }, controls: [...] },
  { ui: { title: 'lighting', accent: '#fbbf24' }, controls: [...] },
  { ui: { title: 'post fx',  accent: '#f43f5e' }, controls: [...] },
];

Config reference

Complete reference for both config formats and all top-level options.

New format (recommended)

The new format keeps defaults inline with controls, making it easier to read and maintain. Use ui for panel-level options and controls for the array of control objects.

{
  ui: {
    title:     'panel',      // panel header text
    accent:    '#00ff88',   // accent color (hex)
    collapsed: false,       // start collapsed?
    closeKey:  'h',         // keyboard toggle key
  },
  controls: [
    { type: 'slider', key: 'speed', default: 50, min: 0, max: 100 },
    { type: 'toggle', key: 'paused', default: false },
  ]
}

Defaults are extracted automatically from the controls array — you don't need a separate state object.

Legacy format

The original format from v1. Fully supported and will not be removed. State is declared separately as an object, and controls are in a layout array.

{
  title:     'panel',
  theme:     { accent: '#00ff88' },
  collapsed: false,
  closeKey:  'h',
  state: {
    speed:  50,
    paused: false,
  },
  layout: [
    { type: 'slider', key: 'speed',  min: 0, max: 100 },
    { type: 'toggle', key: 'paused' },
  ]
}

Real-world usage patterns for common scenarios.

WebGL / Three.js

window.pnlConfigs = [{
  ui: { title: 'scene', accent: '#00aaff', closeKey: 'h' },
  controls: [
    { type: 'text',   label: 'geometry' },
    { type: 'slider', key: 'segments', label: 'segments', default: 32, min: 3, max: 128, step: 1 },
    { type: 'slider', key: 'scale',    label: 'scale',    default: 1,  min: 0.1, max: 3, step: 0.01 },
    { type: 'text',   label: 'material' },
    { type: 'color',  key: 'color',    label: 'color',    default: '#4488ff' },
    { type: 'toggle', key: 'wireframe',label: 'wireframe', default: false },
    { type: 'slider', key: 'roughness',label: 'roughness', default: 0.5, min: 0, max: 1, step: 0.01 },
    { type: 'text',   label: 'animation' },
    { type: 'toggle', key: 'paused',   label: 'paused',    default: false },
    { type: 'slider', key: 'speed',    label: 'speed',     default: 1, min: 0, max: 5, step: 0.01 },
  ]
}];

function animate() {
  requestAnimationFrame(animate);
  const s = window.pnlStates?.[0];
  if (!s) return;
  if (!s.paused) mesh.rotation.y += 0.01 * s.speed;
  mesh.scale.setScalar(s.scale);
  material.color.set(s.color);
  material.wireframe = s.wireframe;
  material.roughness = s.roughness;
}

Canvas 2D

window.pnlConfigs = [{
  ui: { title: 'canvas' },
  controls: [
    { type: 'color',   key: 'bg',     default: '#111122' },
    { type: 'color',   key: 'fill',   default: '#00ff88', modes: ['slider', 'opacity'], onOpacity: v => alpha = v },
    { type: 'slider',  key: 'radius', default: 40, min: 5, max: 200 },
    { type: 'point2d', key: 'pos',    default: { x: 0, y: 0 }, xRange: [-1,1], yRange: [-1,1] },
  ]
}];

let alpha = 1;

function draw() {
  requestAnimationFrame(draw);
  const s = window.pnlStates?.[0]; if(!s) return;
  const cx = canvas.width  / 2 * (1 + s.pos.x);
  const cy = canvas.height / 2 * (1 - s.pos.y);
  ctx.fillStyle   = s.bg;
  ctx.fillRect(0, 0, canvas.width, canvas.height);
  ctx.globalAlpha = alpha;
  ctx.fillStyle   = s.fill;
  ctx.beginPath();
  ctx.arc(cx, cy, s.radius, 0, Math.PI * 2);
  ctx.fill();
}

Changelog

Version history and migration notes.

v2.0

Major update with a new config format, upgraded color picker, improved scrub interactions, and smoother animations.

New in v2

  • New config format — ui + controls with inline defaults, replaces the separate state + layout objects
  • Upgraded color picker — modular modes (box, slider, opacity, rgb) instead of a basic swatch grid
  • Better slider — full-width track with label/value on top, cleaner readout
  • Improved scrub — unified scrub helper, smarter click vs drag detection
  • Smooth collapse — animated max-height transition instead of instant show/hide
  • Tab animation — slide direction based on tab index

Migrating from v1

v1 config format is fully supported — you don't need to migrate. But if you want to use the new format:

// v1 format (still works)
{
  title: 'panel',
  state: { speed: 50 },
  layout: [{ type: 'slider', key: 'speed', min: 0, max: 100 }]
}

// v2 format (recommended)
{
  ui: { title: 'panel' },
  controls: [{ type: 'slider', key: 'speed', default: 50, min: 0, max: 100 }]
}
→ click ▾ or press ` to collapse
